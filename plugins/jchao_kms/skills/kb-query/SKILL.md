# kb-query — 知识库查询与研究

对知识库发起深度查询，合成结构化回答，并将结果归档回知识库形成回馈循环。

## 知识库路径

```
KB_ROOT = /Users/qianli/0-WORKSPACE/50-Knowledge
```

以下所有路径均相对于 `KB_ROOT`。执行任何操作前，先确认该目录存在。

## 触发方式

| 用法 | 行为 |
|------|------|
| `/kb-query <问题>` | 针对问题查询知识库并生成报告 |
| `/kb-query --no-save <问题>` | 仅查询，不归档结果 |

## 流程

### Step 1: 读取索引

读取 `_index/MASTER_INDEX.md` 全文，理解知识库当前覆盖的所有主题和文章。

### Step 2: 定位相关文章

根据用户问题，从 MASTER_INDEX 中筛选相关文章：
- 通过摘要和概念标签匹配
- 通常选取 3-10 篇最相关的文章
- 如有需要，参考 `_index/CONCEPT_MAP.md` 发现跨分类关联

### Step 3: 深度阅读

逐篇读取相关文章的 `index.md`（如有 `index-zh.md` 优先读中文版）。

对于大型查询（涉及 5+ 篇文章），使用并行 subagent 加速阅读：
- 每个 subagent 读取 2-3 篇文章
- 提取与问题相关的关键信息
- 汇总给主 agent 合成

### Step 4: 合成报告

生成结构化 markdown 报告：

```markdown
# <查询标题>

> 基于知识库 N 篇文章的综合分析。
> 生成时间: YYYY-MM-DD

## 核心回答

<直接回答用户问题，2-5 段>

## 详细分析

<按主题分节展开，引用具体文章>

### <主题 1>
...
> 来源：[文章名](路径)

### <主题 2>
...

## 信息来源

| # | 文章 | 分类 | 相关度 |
|---|------|------|--------|
| 1 | [文章名](路径) | 分类/子分类 | 高 |
| 2 | ... | ... | 中 |

## 知识缺口

<如果发现知识库在该主题上的不足，列出：>
- 缺少 XXX 方面的文章
- YYY 概念仅有 1 篇浅述，缺乏深度分析
```

### Step 5: 归档结果（默认行为）

除非使用 `--no-save` 参数，否则将报告归档到 `_derived/` 目录：

```bash
mkdir -p _derived/
```

文件名格式：`_derived/YYYY-MM-DD-<查询简述>.md`

例如：`_derived/2026-04-04-Agent记忆系统对比.md`

### Step 6: 更新 MASTER_INDEX（可选）

如果报告具有长期参考价值（综合了 3+ 篇文章的深度分析），在 MASTER_INDEX.md 末尾的派生内容区追加条目：

```markdown
## _derived

- [Agent记忆系统对比](../_derived/2026-04-04-Agent记忆系统对比.md) — 综合 8 篇文章对比六种记忆架构的优劣 `#记忆系统` `#架构对比`
```

如果 MASTER_INDEX.md 中尚无 `## _derived` 段落，则创建它。

## 查询类型示例

| 查询 | 预期行为 |
|------|---------|
| "Agent 记忆系统有哪些方案？" | 综合 Memory 类 8 篇文章 + 相关架构文章 |
| "autoresearch 方法论的核心是什么？" | 聚焦 Research 类 + Training 类相关文章 |
| "如何从零搭建 API 中转？" | 聚焦 Infrastructure 类 3 篇文章 |
| "知识库里关于投资框架的完整脉络" | 综合 Investment/Fundamentals + Analysis |

## 注意事项

- **不修改原始文章**：仅读取，不编辑已归档的 index.md
- **引用必须真实**：报告中引用的文章路径必须对应真实存在的文件
- **中文输出**：报告始终用中文撰写
- **_derived/ 目录**：不会被 `kb-index-gen.sh` 扫描（不在 DOMAINS 列表中），安全
- **回馈价值判断**：简单查询（如"某篇文章说了什么"）不归档；综合分析才归档
